01

Overview

Tasks Manager is a command-line tool for creating, prioritizing, and automatically executing tasks at scheduled times. Tasks are saved to a JSON file and reloaded on the next startup.

⏱

Time-based Scheduler

Tasks are automatically executed at the desired time — even while the program is running.

💾

JSON Persistence

All tasks are saved to tasks.json and restored on the next startup.

⚡

Immutable Tasks

The Task object is immutable — changes always produce a new instance (functional design pattern).

🔌

Extensible

The Command interface makes it easy to register new commands and add them to the CLI.

ℹ
Work in progress: Some features like --ExecuteAt editing and the GUI interface are not yet fully implemented.

1

Start & Load

Main.java starts — Data.loadFile() loads all tasks from tasks.json into the scheduler.

2

Activate Scheduler

scheduler.start() immediately schedules all loaded tasks in the ScheduledExecutorService.

3

CLI Loop

The user enters commands. CommandRegistry maps the first word to the matching implementation.

4

Save & Exit

On exit, Data.saveFile() is called and the scheduler shuts down cleanly.

03

Running

Prerequisites: Java 21+ and Maven must be installed. Clone the repository and build the project:

🖥 Terminal
Build & Run
# Build the project $ mvn package # Run $ java -cp target/classes cli.Main # Alternatively with IDE (IntelliJ) # → open Main.java → Run > input prompt appears
⚠
Working Directory: The tasks.json file is created in the directory from which you launch the program. Always start from the same folder so your tasks are loaded correctly.

Commands

add
add --<name> -- <priority> -- <time>
✓ stable
Adds a new task and schedules it immediately. The state is automatically saved after adding.

Time Formats

<number>s
Execute in X seconds
<number>min
Execute in X minutes
<number>h
Execute in X hours
Examples
🖥 CLI
# Task "Prepare meeting" with priority 2 in 30 minutes > add --Prepare meeting -- 2 -- 30min # Task "Backup" with highest priority in 2 hours > add --Backup -- 5 -- 2h # Task almost immediately (in 10 seconds) > add --Test run -- 1 -- 10s
list
list
✓ stable
Prints all active tasks — showing UUID, name, priority, and scheduled execution time in the local timezone.
Example Output
🖥 CLI
> list ID : 3f2e1a4b-c9d8-4e7f-a1b2-0c3d4e5f6a7b Name : Prepare meeting Priority : 2 Execution: 2025-06-10T14:30:00+02:00[Europe/Vienna] ID : 8a7b6c5d-4e3f-2a1b-0c9d-8e7f6a5b4c3d Name : Backup Priority : 5 Execution: 2025-06-10T16:00:00+02:00[Europe/Vienna]
remove
remove --<method> <value>
✓ stable
Removes tasks from the scheduler and cancels their scheduled execution. Automatically saves afterwards.

Options

--byName
Removes all tasks with this name (case-sensitive)
--byPriority
Removes all tasks with the given priority (1–5)
--byUUID
Removes exactly the task with this UUID (most precise)
Examples
🖥 CLI
> remove --byName Backup > remove --byPriority 1 > remove --byUUID 3f2e1a4b-c9d8-4e7f-a1b2-0c3d4e5f6a7b
change
change --<field> <uuid> <value>
✓ stable
Changes the name or priority of a task by its UUID. The old task is replaced — the scheduler reschedules the updated task automatically.

Options

--Name
Sets a new name for the task
--Priority
Sets a new priority (1–5)
--ExecuteAt
Not yet implemented (TODO)
Examples
🖥 CLI
# Change name > change --Name 3f2e1a4b-c9d8-4e7f-a1b2-0c3d4e5f6a7b Emergency Backup # Set priority to 5 > change --Priority 3f2e1a4b-c9d8-4e7f-a1b2-0c3d4e5f6a7b 5
sort
sort --<method>
✓ stable
Sorts the internal task list and saves the new order to the JSON file. Affects the output of list.

Options

--byName
Alphabetically by name (A → Z)
--byPriority
Ascending by priority (1 → 5)
--byTime
Chronologically by execution time (earliest first)
Examples
🖥 CLI
> sort --byPriority > sort --byTime > sort --byName
🖥 CLI
Quit program
> exit # Saves tasks.json and shuts down the scheduler cleanly
05

Task Model

A Task is an immutable Java object. It has four fields, and any change produces a new instance via wither methods.

core.Task — Fields
id
UUID
Unique identifier, auto-generated
name
String
Name of the task (not null)
priority
int
Priority from 1 (low) to 5 (critical)
executeAt
Instant
Scheduled execution time (UTC)

Priority Levels

1
Low
2
Normal
3
Medium
4
High
5
Critical
☕ Java
Immutable Pattern
// Create a task Task t = new Task("Backup", 5, Instant.now().plusSeconds(3600)); // Change name → new instance, original unchanged Task t2 = t.withName("Emergency Backup"); // Change priority → yet another new instance Task t3 = t2.withPriority(3);
06

Data Storage

All tasks are stored in a file called tasks.json in the working directory. The format is human-readable JSON.

File Format tasks.json
{ "3f2e1a4b-c9d8-4e7f-a1b2-0c3d4e5f6a7b": { "name": "Prepare meeting", "priority": 2, "executeAt": "2025-06-10T12:30:00Z" }, "8a7b6c5d-4e3f-2a1b-0c9d-8e7f6a5b4c3d": { "name": "Backup", "priority": 5, "executeAt": "2025-06-10T14:00:00Z" } }
✓
The file is automatically updated after every add, remove, sort, and on exit. No manual saving required.
07

Scheduler

The scheduler uses Java's ScheduledExecutorService with a single-thread pool. It schedules each task individually and automatically removes it after execution.

☕ Java
Internal Execution
// What happens when a task fires: executor.schedule(() -> { try { System.out.println("\nEXEC " + task.getName()); // → a Runnable action could run here (TODO) } finally { scheduled.remove(task.getId()); remove(task.getId()); // remove from list } }, delay, TimeUnit.MILLISECONDS);
ℹ
RunnableActions: The folder core/RunnableActions/ already contains ideas — e.g. RandomFact, SpaceFact, and codingChallange. These are planned to be bindable as real actions on tasks in the future.

Thread Safety

All access to the task list and scheduled map is protected with synchronized. The snapshot() pattern prevents ConcurrentModificationException during iteration.
